# /generate-magic Command

**Goal:** Scan all available skills, commands, and plugins, then write the result to `.claude/AVAILABLE_MAGIC.md` for instant display by `/available-magic`.

**Command name:** `/generate-magic`

---

## Prompt

You are generating a static inventory of all skills, commands, and plugins available in this Claude Code session.

**Output file:** `.claude/AVAILABLE_MAGIC.md` (overwrite if it exists)

**Rule:** Do not exclude any commands or skills. Surface everything discovered from plugins, the skill registry, local `.claude/commands/`, and local `.claude/skills/`.

---

## 1. Gather Available Magic

**Run in parallel:**

1. Run `claude plugin list` to get installed plugins
2. Parse the available skills from the Skill tool's registry (you have access to this in your system prompt)
3. **Scan `.claude/commands/`** — Walk the directory to discover all project slash commands.
   - Focus on `otl/` (util, orch, prds, sop) and any sibling dirs (bmad, opsx, etc.)
   - Read each command file's frontmatter for `name`, `description`, `category`
   - Include everything found; do not exclude any commands
4. **Scan `.claude/skills/`** — Walk the directory to discover all local skill definitions.
   - Each skill is a directory containing a `SKILL.md` file (e.g., `.claude/skills/morning-briefing/SKILL.md`)
   - Read each `SKILL.md` frontmatter for `name`, `description`
   - These are locally installed skills that may not appear in the system prompt's skill registry
   - Cross-reference with the skill registry (step 2) to avoid duplicates — if a skill appears in both, use the local `SKILL.md` frontmatter as the authoritative source
   - Include standalone files like `.claude/skills/kenwood-blog-README.md` as reference docs, not skills

---

## 2. Write `.claude/AVAILABLE_MAGIC.md`

Write the complete inventory as a markdown file. **Plugins and Quick Reference appear FIRST, then Skills & Commands.**

Group by module. **Presentation order:**

1. **Installed Plugins** (FIRST — quick context on what's wired in)
2. **Quick Reference** (how to invoke skills, regenerate, get help)
3. **otl:\*** — Otage Labs tools and commands
4. kenwood-* — Kenwood content pipeline (blog pipeline + variant generation, from `.claude/skills/`)
5. frontend-design:* — Frontend design generation
6. Other local skills from `.claude/skills/` (morning-briefing, remotion-best-practices, etc.)
7. Other system-prompt skills (update-config, keybindings-help, simplify, loop, schedule, claude-api, find-skills)
8. opsx:* — OpenSpec change management
9. bmad:* — BMAD framework (group by bmb, bmm, core, and standalone) (LAST)
