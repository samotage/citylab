# /generate-codex Command

**Goal:** Bootstrap the Codex runtime by translating Claude Code artifacts (commands, skills, rules) into their Codex equivalents. Shared artifacts go into `otl_support/python/.codex/`; per-project AGENTS.md files are compiled for each Python project.

**Command name:** `/generate-codex`

---

## Prompt

You are generating Codex runtime artifacts from the Claude Code source of truth. This is a two-phase process: Phase 1 generates shared artifacts (prompts and skills), Phase 2 compiles per-project AGENTS.md files.

**Source of truth:** `.claude/commands/`, `.claude/skills/`, `.claude/rules/`, `CLAUDE.md`
**Shared output:** `otl_support/python/.codex/prompts/`, `otl_support/python/.codex/skills/`
**Per-project output:** `AGENTS.md` in each Python project root

**Rule:** Translate all discovered commands and skills. Do not filter or exclude. Report anything skipped with a reason.

---

## Phase 1: Shared Artifacts

These outputs land in `otl_support/python/.codex/` and are shared across all Python projects via the existing `.codex/` symlink.

### 1a. Prompts (from `.claude/commands/`)

Walk `.claude/commands/` recursively. For each `.md` file:

1. Read the file and parse YAML frontmatter
2. **Translate frontmatter** to Codex prompt format:
   - Keep `description` as-is
   - If the command body mentions accepting an argument (e.g., "The argument after `/command-name` is..."), add `argument-hint` with a short usage hint
   - Drop `name`, `id`, `category`, `tags` — Codex prompts don't use these fields
3. **Flatten the path** into the filename: `otl/util/generate-magic.md` → `otl--util--generate-magic.md`
   - Use `--` as the separator (double dash)
   - Files already at the top level keep their name: `opsx/new.md` → `opsx--new.md`
4. Write the translated file to `otl_support/python/.codex/prompts/`
5. **Skip** `README.md` files (they're documentation, not commands)

**Cleanup:** After generating all prompts, scan the target directory for any `.md` files that no longer have a corresponding source command. Delete orphans and report them.

### 1b. Skills (from `.claude/skills/`)

Walk `.claude/skills/`. For each directory containing a `SKILL.md`:

1. Read the existing `SKILL.md` and parse its frontmatter
2. The Claude Code SKILL.md format is already compatible with Codex — **copy as-is**
   - Both formats use `name`, `description`, and optional `argument-hint`
   - Codex additionally supports `metadata.short-description` — add this field if missing, using the first sentence of `description`
3. Copy the entire skill directory (SKILL.md + any supporting files) to `otl_support/python/.codex/skills/{skill-name}/`
4. **Skip** standalone `.md` files that are not inside a skill directory (e.g., `kenwood-blog-README.md`)
5. **Skip** symlinked skill directories (e.g., `remotion-best-practices` → `../../.agents/skills/...`) — log a warning

**Cleanup:** After generating all skills, scan the target directory for any skill directories that no longer have a corresponding source. Delete orphans and report them.

---

## Phase 2: Per-Project AGENTS.md

Compile an AGENTS.md for each Python project. AGENTS.md is Codex's equivalent of CLAUDE.md — it's the project instruction file that Codex reads at startup. Because Codex has no project-level rules directory, rules must be inlined.

### Target projects

**Discover dynamically** — do NOT hardcode project paths. Run:

```bash
cli-headspace projects list
```

Parse the JSON response. Filter to projects where `primary_language == "python"`. Use the `path` field from each matching project as the directory.

**If a project's `path` does not exist on this machine, skip it with a warning. Do not fail.**

### Compilation per project

For each project that exists:

1. **Read `CLAUDE.md`** from the project root. If missing, skip the project with a warning.
2. **Read all `.md` files from `.claude/rules/`** in the project (walk recursively, sorted alphabetically). These include both shared rules (via symlink from otl_support) and any project-local rules.
3. **Compile AGENTS.md** with this structure:

```markdown
# AGENTS.md
# Auto-generated by /generate-codex on {YYYY-MM-DD}
# Source: CLAUDE.md + .claude/rules/
# Regenerate: /generate-codex
# DO NOT EDIT — changes will be overwritten

{contents of CLAUDE.md}

---

# Compiled Rules

{for each rules file, sorted by path:}

## {filename without .md extension}

{file contents}

{end for}
```

4. **Write `AGENTS.md`** to the project root. **CRITICAL:** If AGENTS.md is a symlink (e.g., to CLAUDE.md), remove the symlink first before writing — otherwise `open()` follows the symlink and overwrites the target file. Always verify the written file is a regular file, not a symlink.
5. Report the project name, file size, and number of rules files compiled.

---
